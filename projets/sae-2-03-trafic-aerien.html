01 — Contexte

Contexte du projet

Cette SAE (Situation d'Apprentissage et d'Évaluation) nous a été confiée en équipe de trois (avec Jordan et Yacine) durant le second semestre de 1ère année de BUT R&T. L'objectif : développer une application web de gestion du trafic aérien entre aéroports, avec un schéma de données couvrant aéroports, pistes d'atterrissage, compagnies aériennes, types d'avions, avions et vols.

Le cahier des charges imposait un CRUD complet pour chaque entité, une logique métier d'attribution automatique de piste (vérification de compatibilité piste/avion, blocage de 10 minutes par piste autour de chaque atterrissage, proposition d'un créneau +15 minutes en cas d'indisponibilité), un import de vols par fichier CSV, ainsi qu'une fiche récapitulative des vols par aéroport et par période.

Nous avons choisi une stack Python/Django avec une base MariaDB, déployée sur deux machines virtuelles Debian (une pour l'application via Nginx + Gunicorn, une pour la base de données) — une organisation proche d'un déploiement de production réelle.

Extrait de code — la fonction trouver_piste_libre() que j'ai conçue (gestion/views.py) :

# Cette fonction cherche une piste disponible pour l'atterrissage.
# Le sujet demande qu'une piste soit bloquée 10 minutes autour de chaque atterrissage.
# On parcourt toutes les pistes de l'aéroport d'arrivée qui sont assez longues
# pour le type d'avion, et on vérifie qu'aucun autre vol n'arrive au même moment.
# Si toutes les pistes sont prises, on suggère 15 minutes plus tard.
def trouver_piste_libre(aeroport_arrivee, type_avion, heure_arrivee):

    # Pistes dont la longueur est suffisante pour ce type d'avion
    pistes_assez_longues = Piste.objects.filter(
        aeroport=aeroport_arrivee,
        longueur__gte=type_avion.longueur_piste_necessaire
    )

    if not pistes_assez_longues:
        return None, None

    for piste in pistes_assez_longues:
        debut_occupation = heure_arrivee - timedelta(minutes=10)
        fin_occupation = heure_arrivee + timedelta(minutes=10)

        vols_sur_cette_piste = Vol.objects.filter(
            piste_arrivee=piste,
            datetime_arrivee__gte=debut_occupation,
            datetime_arrivee__lte=fin_occupation
        )

        if not vols_sur_cette_piste:
            return piste, heure_arrivee

    # Aucune piste n'est libre : on propose un créneau 15 minutes plus tard
    heure_suggeree = heure_arrivee + timedelta(minutes=15)
    return None, heure_suggeree

05 — English summary

Project summary (English)

As part of a 3-person team project for our first-year Network & Telecommunications degree (BUT R&T, IUT de Colmar), we built a Django web application to manage air traffic between airports, with full CRUD operations for airports, runways, airlines, aircraft types, aircraft, and flights, backed by a MariaDB database and deployed across two Debian virtual machines (Nginx + Gunicorn for the app, MariaDB for the database).

My personal contributions were the initial Django setup, the full database schema and migrations, the CRUD for Airports and Aircraft Types, the CSV bulk-import feature for flights, and — the most critical part — the runway-allocation algorithm, which checks whether a runway is long enough for a given aircraft type and free within a ±10-minute window around the landing time, suggesting a +15-minute delay if no runway is available. I also set up the deployment virtual machine and helped write the installation procedure.

This project strengthened my skills in relational database design, Django development, business-logic algorithms, and basic system administration/deployment — directly relevant to my goal of working at the intersection of infrastructure, automation and client-facing tools.
